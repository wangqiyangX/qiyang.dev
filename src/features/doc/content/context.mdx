## Features

- Composes context usage through a compound component API.
- Shows context window utilization with a circular trigger indicator and progress bar.
- Breaks usage into input, output, reasoning, and cached tokens.
- Estimates per-row and total cost with `tokenlens`.
- Reads AI SDK `LanguageModelUsage` details, including the v6 token detail fields.
- Uses shadcn/ui `Button`, `HoverCard`, and `Progress` primitives.

## Usage

```tsx
import {
  Context,
  ContextCacheUsage,
  ContextContent,
  ContextContentBody,
  ContextContentFooter,
  ContextContentHeader,
  ContextInputUsage,
  ContextOutputUsage,
  ContextReasoningUsage,
  ContextTrigger,
} from "@/components/context"
```

```tsx
<Context
  maxTokens={128_000}
  modelId="openai:gpt-4o-mini"
  usedTokens={40_000}
  usage={usage}
>
  <ContextTrigger />
  <ContextContent>
    <ContextContentHeader />
    <ContextContentBody>
      <ContextInputUsage />
      <ContextOutputUsage />
      <ContextReasoningUsage />
      <ContextCacheUsage />
    </ContextContentBody>
    <ContextContentFooter />
  </ContextContent>
</Context>
```

## Component Architecture

1. `<Context>` provides token usage data to every child.
2. `<ContextTrigger>` renders the interactive trigger.
3. `<ContextContent>` renders the hover card container.
4. `<ContextContentHeader>` renders progress and token totals.
5. `<ContextContentBody>` groups usage rows.
6. `<ContextContentFooter>` renders total cost.
7. Usage components render the individual token categories.

## Token Formatting

Token counts use `Intl.NumberFormat` with compact notation, so values scale from
exact counts to `K`, `M`, and `B` suffixes automatically.

## Cost Calculation

When `modelId` is provided, the component passes non-cached input, output,
reasoning, cache reads, and cache writes to `tokenlens`. Costs are formatted as
USD and omitted when pricing is unavailable.
