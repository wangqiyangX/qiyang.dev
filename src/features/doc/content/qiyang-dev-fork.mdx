This site is a personal fork of the original portfolio, blog, and shadcn registry project. The goal of this fork is to keep the original structure intact while replacing the public identity, profile data, project links, and site metadata with information for Qiyang Wang.

## What This Fork Keeps

- Next.js App Router architecture
- MDX-powered content system
- Portfolio, blog, and registry sections
- shadcn/ui and Tailwind CSS v4 styling
- SEO, RSS, sitemap, and AI-readable routes

## What This Fork Changes

- Site identity now points to qiyang.dev
- Portfolio data is rewritten around Qiyang Wang
- Social links and source links point to Qiyang's public profiles
- Original personal awards, testimonials, sponsors, and private contact details are removed until confirmed
- Blog discovery only includes content explicitly marked with `category: blog`

## Project Direction

qiyang.dev is being shaped into a home for personal reading notes, software projects, reusable UI work, and small experiments. The first pass focuses on replacing owner-specific information without changing the upstream project layout, so future updates can continue to reuse the existing data, content, and registry pipeline.
